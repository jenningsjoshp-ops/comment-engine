This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Start dev server (opens QR code for Expo Go)
npx expo start

# Run on specific platform
npx expo start --ios
npx expo start --android

# Build for production (iOS)
NODE_OPTIONS=--use-system-ca eas build --profile production --platform ios

# Submit to App Store
NODE_OPTIONS=--use-system-ca eas submit --platform ios
```

> `NODE_OPTIONS=--use-system-ca` is required on this Windows machine for EAS commands to reach Expo's servers — omitting it causes TLS certificate errors.

The `eas-build-pre-install` script (`generate-config.js`) runs automatically before EAS builds to write `config.js` from environment variables (`ANTHROPIC_API_KEY`, `APIFY_API_TOKEN`). For local dev, `config.js` has keys hardcoded directly — do not overwrite it with the generator script manually.

## Architecture

**Entry point:** `index.js` → `App.js`

**App.js** is the single source of truth for all global state. It owns:
- User session (`userId`, `userProfile`, `isOnboarded`, `showWelcome`)
- Tier/limits (`tier`, `tierLimits`, `discoveryLimits`)
- Comment history (`commentHistory`, `selectedComments`, `commentCount`)
- Discovery tracking (`discoveryCount`, `lastDiscoveryDate`, `discoveryCache`)
- Engaged accounts (`engagedAccounts`, `commentedPostUrls`)

All state is passed down as props — there is no context or state management library. The navigation flow is: `WelcomeScreen` → `OnboardingScreen` → main stack (`Main`, `Discover`, `Settings`, `Reporting`, `Feedback`).

**Screens (`screens/`)** are all self-contained functional components receiving props from App.js:

- **MainScreen** — paste an Instagram post URL → Apify scrapes the caption → Anthropic generates 3 comments → user taps to copy + auto-opens Instagram. Two-phase loading state (`fetch` / `generate`) drives the `LoadingMessages` animation component.
- **DiscoverScreen** — uses Apify to find posts by hashtag, queues them for commenting. Has its own `selectedPost` state and `completedPosts`/`skippedPosts` tracking. Calls `onCommentUsed` and `onDiscoveryUsed` callbacks up to App.js.
- **OnboardingScreen** — multi-step flow collecting email, name, IG handle, account type, voice sliders, hashtags, and up to 5 recent IG post URLs. Calls `onComplete(profile)` when done.
- **SettingsScreen** — edits the user profile in place, supports tier upgrades.
- **ReportingScreen** — read-only stats view using `commentHistory` and `engagedAccounts` passed from App.js.

**`lib/supabase.js`** — all database I/O. Tables used: `users`, `comment_history`, `commented_posts`, `engaged_accounts`, `discovery_cache`, `feedback`. The client is initialized with a hardcoded public anon key (safe for client-side). Functions follow a consistent pattern: upsert for creates/updates, return `null` on error rather than throwing.

**`config.js`** — exports `ANTHROPIC_API_KEY` and `APIFY_API_TOKEN`. In production/EAS builds this file is generated by `generate-config.js` from env vars. In local dev it contains the keys directly.

## Key Patterns

**AI comment generation** (`MainScreen`, `DiscoverScreen`): Both screens call the Anthropic API directly from the client using `anthropic-dangerous-direct-browser-access: true`. The system prompt is built from `userProfile` fields — `accountType`, `sliders`/`sliderValues`, `igPosts`, and `referenceUrls` — to match the user's voice. The last 10 selected comments are injected as learning context when ≥3 exist.

**Voice sliders**: `userProfile.sliders` is an array of `{id, left, right}` objects. `userProfile.sliderValues` maps `id → number (1-5)`. Both arrays must stay in sync — when editing sliders in Settings, update both.

**Tier limits**: Enforced client-side only. `tierLimits` caps `commentCount` (monthly comments), `discoveryLimits` caps daily Discover uses reset by date comparison against `lastDiscoveryDate`.

**Monthly comment count**: `commentCount` tracks only the current calendar month. On startup, `checkExistingUser` in App.js filters the loaded comment history to `month === now.getMonth() && year === now.getFullYear()` before calling `setCommentCount`. New comments incremented via `handleCommentUsed` during the session are always this month, so no extra filtering needed there. Do not revert this to `history.length` — that would count all-time comments against the monthly cap.

**`selectComment` pattern** (both MainScreen and DiscoverScreen): copies to clipboard → shows "Copied!" for 1 second via `setCopiedIndex` → clears state and auto-opens Instagram with `Linking.openURL`. No alert popups.

**Voice rules** enforced in the AI system prompt — never break these when editing prompts:
- 1–2 sentences max
- Never use dashes or em dashes or hyphens between words
- Sounds like a text, not a caption
- Never promotional
- Makes people curious about the commenter

**Discovery filters** (applied in DiscoverScreen when queuing posts): recent posts only, 50–10,000 likes, under 50 comments, accounts under 100K followers (`ownerFollowersCount`). Never surface a post whose URL is in `commentedPostUrls`.

**Discovery cache** is two-layer: in-memory state (`discoveryCache` from App.js) checked first, then Supabase `discovery_cache` table (24h TTL, keyed by `today:hashtag1,hashtag2,...`). Only a full cache miss hits Apify and consumes a daily session. The daily limit check (`discoveryRemaining <= 0`) only blocks the Apify path — cache hits are always free.

**Feedback** is saved to the `feedback` Supabase table via `saveFeedback()` in `lib/supabase.js`. The table has RLS enabled with an anon-insert policy. Schema: `id, user_name, email, ig_handle, account_type, type, message, created_at`.

## Git Rules

`config.js` must never be committed — it contains live API keys. If it gets tracked:
```bash
git rm --cached config.js
```

If a push is rejected because secrets were detected in history, use the orphan branch approach (rewrite history). Always push with:
```bash
git push origin master:main
```
